Проект: https://parts-kronos5.ru/  
Окружение по умолчанию: **Chrome (desktop)**; для SMS — **iOS (Messages)**.  

---

## Стандарт оформления
Порядок секций внутри баг-репорта:
1) Метаданные (Project/Component/Environment/URL/Build/Severity/Priority/Reproducibility)  
2) Preconditions  
3) Test Data  
4) Steps to reproduce  
5) Actual result  
6) Expected result  
7) Attachments  

---

## Severity / Priority

**Severity (impact):**
- **Blocker** — полностью блокирует ключевой путь (оформление/оплата/авторизация) без обхода.
- **Critical** — ключевой путь сильно нарушен.
- **Major** — значимое ухудшение конверсии/UX (поиск, каталог, корзина), путь не всегда блокируется.
- **Minor** — верстка/текст, не мешает завершить сценарий.
- **Trivial** — мелкие баги без влияния.

**Priority (urgency):**
- **P0** — первостепенно.
- **P1** — высокий: конверсия/ключевые сценарии/массово воспроизводится.
- **P2** — средний: локально/есть обход.
- **P3** — низкий: верстка/текст/можно отложить.
