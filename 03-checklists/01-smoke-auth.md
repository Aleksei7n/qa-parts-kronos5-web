# Smoke checklist — Auth (Chrome, desktop)

## Login
- [ ] Модалка “Войти” открывается/закрывается (крестик/ESC/клик вне)
- [ ] Поля отображаются: **Email** и **Пароль**
- [ ] Кнопка “Войти” disabled при пустом Email
- [ ] Кнопка “Войти” disabled при пустом Пароле
- [ ] “Войти” становится enabled при заполненных Email + Пароль
- [ ] Неверные данные → ошибка “Неверный логин или пароль”, авторизация не происходит
- [ ] Ссылки “Регистрация” и “Забыли пароль? Восстановить” видимы и кликабельны

## Password recovery
- [ ] Переход по “Забыли пароль? Восстановить” открывает форму восстановления
- [ ] В форме восстановления есть поле **Email**
- [ ] Невалидный email → сообщение об ошибке, отправка недоступна/не выполняется
- [ ] Валидный email → запрос отправляется, показывается подтверждение (success state)

## Registration
- [ ] Открывается форма регистрации
- [ ] Поля видимы: ФИО, Email, Телефон (выпадающий список с выбором страны) (+7/+375), Пароль, Повтор пароля
- [ ] “Получить код” disabled при пустом/невалидном телефоне
- [ ] Валидация email: невалидный → ошибка “Введите корректную почту”
- [ ] Пароль < 8 → ошибка/подсказка, регистрация недоступна
- [ ] Пароли не совпадают → ошибка/подсказка “Пароли не совпадают”
- [ ] Ввод корректного телефона → “Получить код” enabled, отправка кода инициируется
- [ ] Некорректный SMS-код → регистрация недоступна / ошибка/подсказка “Неверный SMS-код”
- [ ] Корректные данные + согласие → “Зарегистрироваться” enabled
- [ ] Успешная регистрация → “Регистрация прошла успешно” + переход на форму входа
